# Roadmap

This roadmap describes the next useful steps for building FableCodex as an OSS ecosystem. It is not a promise of dates.

## Now

- Keep core tests green for scripts, manifests, license metadata, and coverage matrix parsing.
- Improve contributor onboarding with issue templates, pull request templates, and governance docs.
- Keep README installation and usage paths accurate for local marketplace installs.

## Next

- Add release checklist automation for plugin packaging.
- Add examples for common workflows: debugging, frontend verification, prompt conversion, provider bridge setup, and long-running goal ledgers.
- Add a small changelog process.
- Add fixture-based coverage tests when a redistributable mock source prompt is available.

## Later

- Publish curated community examples for Fable-style Codex workflows.
- Add compatibility checks for future Codex plugin manifest changes.
- Add optional docs for using GitHub, Drive, Figma, Notion, and Browser connectors with this workflow style.
- Consider a maintainers guide once there are recurring contributors.

## Out Of Scope

- Reproducing model weights, hidden safety systems, or provider-specific runtime behavior.
- Shipping credentials or provider access.
- Reconstructing proprietary or leaked prompts.
